# 🛡️ Защита от бана Avito

## Что сделано для защиты

### 1. Ротация User-Agent
При каждом запросе используется **случайный User-Agent** из 7 вариантов:
- Chrome (разные версии)
- Firefox (разные версии)
- Safari на Mac

### 2. Рандомизация поведения
- **Размер окна**: Меняется случайно (1920x1080, 1680x1050, 1440x900, 1366x768)
- **Движения мыши**: Случайное количество (2-5 движений) в случайные точки
- **Задержки**: Варьируются от 1.5 до 3.5 секунд
- **Скорость прокрутки**: Рандомная

### 3. Задержки между батчами
По умолчанию **3000ms** между батчами (настраивается в `.env`)

### 4. Очистка cookies (опционально)
Можно включить очистку cookies при каждом запуске

## ⚙️ Настройка защиты

### Файл `parser/.env`:

```env
# Порт сервера
PORT=3300

# Максимум страниц одновременно (меньше = безопаснее)
MAX_PAGES=3

# Замедление симуляции (больше = естественнее)
SIMULATION_SLOWDOWN=2.5
SIMULATION_TIMEOUT_MS=12000

# Очистка cookies при запуске (1 = включить, 0 = выключить)
CLEAR_BROWSER_DATA=0
```

### Файл `detail-parser/.env`:

```env
DATABASE_URL=postgres://user:pass@localhost:5433/db_name
PARSER_BASE_URL=http://localhost:3300

# Размер батча (меньше = безопаснее)
BATCH_SIZE=3

# Задержка между батчами (больше = безопаснее)
BATCH_DELAY_MS=3000
```

## 📊 Рекомендации по настройкам

### Если парсите много (1000+ объявлений):

**АГРЕССИВНАЯ защита (медленно, но безопасно):**
```env
# parser/.env
MAX_PAGES=1
SIMULATION_SLOWDOWN=3.0
CLEAR_BROWSER_DATA=1

# detail-parser/.env
BATCH_SIZE=2
BATCH_DELAY_MS=5000
```

**Время**: ~2-3 часа на 1000 объявлений

---

**СБАЛАНСИРОВАННАЯ защита (текущие настройки):**
```env
# parser/.env
MAX_PAGES=3
SIMULATION_SLOWDOWN=2.5
CLEAR_BROWSER_DATA=0

# detail-parser/.env
BATCH_SIZE=3
BATCH_DELAY_MS=3000
```

**Время**: ~1-1.5 часа на 1000 объявлений

---

**БЫСТРАЯ обработка (риск бана выше):**
```env
# parser/.env
MAX_PAGES=5
SIMULATION_SLOWDOWN=1.5
CLEAR_BROWSER_DATA=0

# detail-parser/.env
BATCH_SIZE=5
BATCH_DELAY_MS=1500
```

**Время**: ~30-40 минут на 1000 объявлений
**⚠️ Риск**: Может забанить при больших объемах

## 🚨 Что делать если забанили

### ♻️ Автоматический перезапуск (встроено)

Парсер **автоматически** обнаруживает блокировку и перезапускается! 

**Что происходит при бане:**
1. Обнаружение блокировки по ключевым фразам на странице
2. Автоматическая очистка cookies в браузере
3. Ожидание 30 минут (настраивается)
4. Продолжение парсинга с того же места

**Настройки в `detail-parser/.env`:**
```env
# Количество попыток перезапуска
MAX_BAN_RETRIES=3

# Задержка между попытками (30 минут)
BAN_RETRY_DELAY_MS=1800000
```

**⚠️ ВАЖНО:** Автоперезапуск НЕ меняет IP адрес!
- Используйте прокси с ротацией IP (см. `PROXY-SETUP.md`)
- Или вручную меняйте IP через VPN

---

### Ручная обработка (если автоперезапуск не помог)

### 1. Очистить cookies
Установите в `parser/.env`:
```env
CLEAR_BROWSER_DATA=1
```

Или вручную удалите папку:
```bash
rm -rf parser/browser-data
```

### 2. Увеличить задержки
```env
BATCH_DELAY_MS=10000  # 10 секунд между батчами
SIMULATION_SLOWDOWN=4.0
```

### 3. Уменьшить параллелизм
```env
MAX_PAGES=1
BATCH_SIZE=1
```

### 4. Использовать прокси
См. подробную инструкцию в файле `PROXY-SETUP.md`

Краткая версия - добавьте в `parser/.env`:
```env
PROXY_SERVER=your-proxy:port
PROXY_USERNAME=username
PROXY_PASSWORD=password
```

## 💡 Дополнительные советы

1. **Парсите ночью** (меньше нагрузка на Avito)
2. **Разбивайте на сессии**: парсите по 200-300 объявлений, делайте перерыв 30-60 минут
3. **Используйте VPN/прокси** для смены IP между сессиями
4. **Не парсите слишком часто** - 1-2 раза в день максимум
5. **Мониторьте ответы**: если начали приходить капчи - остановитесь

## 🔍 Признаки бана

- Появляется капча на страницах
- Возвращается пустой HTML
- Ошибка 403 Forbidden
- Редирект на страницу проверки

Если увидели эти признаки - **остановите парсер**, очистите cookies, увеличьте задержки, смените IP.
