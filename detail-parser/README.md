# Detail Parser

Парсер для получения описаний объявлений с Avito.

## Установка

```bash
npm install
```

## Настройка

Отредактируйте `.env` файл:

```env
DATABASE_URL=postgres://user:password@localhost:5432/dbname
PARSER_BASE_URL=http://localhost:3300
BATCH_SIZE=2
BATCH_DELAY_MS=5000
MAX_BAN_RETRIES=3
BAN_RETRY_DELAY_MS=1800000
```

### Параметры

- `DATABASE_URL` - строка подключения к PostgreSQL
- `PARSER_BASE_URL` - адрес Puppeteer сервера (должен совпадать с портом в `parser/.env`)
- `BATCH_SIZE` - количество URL для одновременной обработки (рекомендуется 1-2)
- `BATCH_DELAY_MS` - задержка между батчами в миллисекундах (рекомендуется 5000+)

### Автоперезапуск при бане

- `MAX_BAN_RETRIES` - максимальное количество попыток после обнаружения бана (по умолчанию: 3)
- `BAN_RETRY_DELAY_MS` - задержка перед повторной попыткой в миллисекундах (по умолчанию: 1800000 = 30 минут)

При обнаружении блокировки IP парсер автоматически:
1. Очищает cookies в браузере
2. Ждет указанное время (30 минут по умолчанию)
3. Продолжает парсинг с того же места

После исчерпания всех попыток парсер останавливается и выводит инструкции по устранению блокировки.

## Запуск

Из корневой папки проекта:

```bash
npm start
```

Это запустит одновременно:
- Puppeteer сервер (`parser/`)
- Detail parser (этот парсер)

## Структура БД

Парсер ожидает таблицу `avito_items` со следующими столбцами:
- `avito_item_id` - ID объявления (первичный ключ)
- `url` - ссылка на объявление
- `status` - статус объявления (обрабатываются только записи со значением 'active')
- `description` - описание объявления (заполняется парсером)

## Антибан меры

Для минимизации риска блокировки:
- Используйте `BATCH_SIZE=1` или `2` (не больше)
- Устанавливайте `BATCH_DELAY_MS` от 5000 до 10000 мс
- Настройте прокси с ротацией IP в `parser/.env`
- Используйте случайные User-Agent (встроено в Puppeteer сервер)
- Включайте `CLEAR_BROWSER_DATA=1` в `parser/.env` для очистки cookies между запусками

Подробнее см. `ANTI-BAN.md` и `PROXY-SETUP.md` в корне проекта.
